# Ch02: 변수와 타입 — 대화형 튜토리얼

---

## 1. 변수는 기본이 불변

> **선생님**: Rust에서 변수를 만들면 **기본적으로 값을 못 바꿔요.**
>
> ```rust
> let x = 5;
> x = 10;  // 컴파일 에러!
> ```
>
> C++에서는 `int x = 5; x = 10;` 이게 당연한데, Rust는 막아요.
>
> **왜?** → 값이 중간에 바뀌면 버그 추적이 어렵거든요. 특히 멀티스레드에서. 그래서 "바꿀 거면 명시적으로 말해" 라는 철학이에요.
>
> ```rust
> let mut x = 5;  // "나 이거 바꿀 거야" 선언
> x = 10;          // OK
> ```

**🙋 학생**: 오 이런 스토리텔링 좋은 것 같아. 일단 멀티스레드에서 쓰려고 만든건가 그럼?

> **선생님**: 멀티스레드 **때문에** 만든 건 아니고, Rust의 전체 철학이 **"실수를 컴파일 타임에 잡자"**예요. 그중 하나가 불변 기본값인 거고, 이게 멀티스레드에서도 자연스럽게 안전해지는 부수 효과가 있는 거예요.
>
> 지금 단계에서는 깊이 안 들어가도 됩니다. **"Rust는 기본이 못 바꿈, 바꾸려면 mut"** 이것만 가져가면 돼요.

### 💡 핵심
- Rust 철학 = "실수를 컴파일 타임에 잡자"
- 변수는 기본 **불변**, 바꾸려면 `mut` 명시

---

## 2. 타입은 대부분 안 써도 된다

> **선생님**: `let x = 5;` 이렇게 쓰면 Rust가 알아서 "아 이건 정수구나" 하고 `i32`로 정해요. 아까 궁금했던 `u32` 같은 건 **내가 직접 타입을 지정할 때** 쓰는 거예요.
>
> ```rust
> let x = 5;           // Rust가 알아서 i32
> let y: u32 = 5;      // 내가 "양수만 쓸 거야" 지정
> ```
>
> 대부분은 Rust가 알아서 추론해주니까, **타입을 안 써도 됩니다.** 써야 하는 경우는 나중에 만나면 그때 알려드릴게요.

### 💡 핵심
- 타입 추론: Rust가 알아서 판단 (`let x = 5;` → i32)
- 직접 지정도 가능: `let y: u32 = 5;`
- 대부분 안 써도 OK

### 📝 타입 읽는 법
- `i` = signed (음수 가능), `u` = unsigned (양수만), `f` = float (소수점)
- 뒷숫자 = 비트 수 (메모리 크기)
- 자주 쓰는 것: `i32` (기본 정수), `f64` (기본 실수), `bool`, `&str`

## 3. 섀도잉 — 같은 이름으로 새 변수 만들기

> **선생님**: `let`으로 만든 변수는 **같은 이름으로 다시 만들 수 있어요.** 이걸 **섀도잉**이라고 해요.
>
> ```rust
> let x = 5;
> let x = x + 1;   // x는 이제 6
> let x = x * 2;   // x는 이제 12
> ```
>
> "어? mut 안 썼는데 값이 바뀌었잖아?" → 아뇨, 바뀐 게 아니라 **새 변수를 만든 거예요.** 이전 x는 사라지고 새 x가 생긴 거.
>
> mut과의 차이:
>
> ```rust
> // mut: 같은 변수의 값을 바꿈
> let mut x = 5;
> x = 10;          // 같은 상자에 값만 교체
>
> // 섀도잉: 새 변수를 만듦
> let x = 5;
> let x = "hello"; // 타입까지 바꿀 수 있음! (새 상자니까)
> ```
>
> mut은 타입을 바꿀 수 없지만, 섀도잉은 타입도 바꿀 수 있어요. 왜냐면 완전히 새 변수니까요.

**🙋 학생**: let이랑 mut이 뭐의 줄임말이죠?

> **선생님**: **let** = "let" (그대로 영어 "~하게 하다", 수학에서 "let x = 5"처럼 "x를 5로 놓자")
> **mut** = "mutable"의 줄임말 (변경 가능한)

### 💡 핵심
- 섀도잉 = `let`으로 같은 이름 재선언 (새 변수)
- mut과 다름: 섀도잉은 **타입도 변경 가능**
- let = "let" (수학 표현), mut = "mutable" (변경 가능)

## 4. 상수(const) vs 불변 변수(let)

> **선생님**: 둘 다 못 바꾸는 건 같은데, 핵심 차이는 **값이 언제 정해지느냐**예요.
>
> ```rust
> let x = some_function();       // OK — 프로그램 실행해봐야 값을 앎
> const MAX: u32 = 100;          // OK — 빌드할 때 이미 100으로 확정
> const Y: u32 = some_function(); // 에러! — 빌드 시점에 값을 모름
> ```
>
> **`const`는 빌드할 때 값이 확정되어야 해요.** 그래서 타입도 반드시 써줘야 하고, 이름도 `대문자_스네이크`가 관례예요.
>
> 쉽게 말하면: `const`는 "절대 변하지 않는 설정값", `let`은 "한번 정하면 안 바뀌는 변수".

### 💡 핵심
- `const` = 빌드 타임에 확정, 타입 필수, 대문자_스네이크
- `let` = 런타임에 정해져도 OK, 타입 생략 가능
- `const`는 설정값, `let`은 변수

## 5. 튜플과 배열

> **선생님**: Rust에서 여러 값을 묶는 방법이 두 가지 있어요.
>
> **튜플** — 타입이 달라도 묶을 수 있음:
>
> ```rust
> let person = ("철수", 25, true);  // 이름, 나이, 활성 여부
> println!("{}", person.0);          // "철수" — 점(.)과 번호로 접근
> println!("{}", person.1);          // 25
> ```
>
> **배열** — 같은 타입만, 길이 고정:
>
> ```rust
> let scores = [90, 85, 77, 92];
> println!("{}", scores[0]);         // 90 — 대괄호[]로 접근
> println!("{}", scores.len());      // 4
> ```
>
> 차이를 한 줄로: **튜플은 다른 타입 OK, 배열은 같은 타입만.**
>
> C++의 `std::tuple`이랑 배열이랑 같은 개념이에요. 근데 Rust 배열은 길이가 고정이라서, 가변 길이가 필요하면 나중에 배울 `Vec`을 써요.

### 💡 핵심
- 튜플: 다른 타입 묶기 OK, `점.번호`로 접근
- 배열: 같은 타입만, 길이 고정, `대괄호[번호]`로 접근
- 가변 길이 필요하면 → `Vec` (나중에 배움)

## Ch02 총정리

| 개념 | 한 줄 요약 |
|------|-----------|
| `let` | 불변 변수 (기본) |
| `let mut` | 가변 변수 (명시적으로 선언) |
| `const` | 상수 (빌드 타임 확정, 타입 필수) |
| 섀도잉 | `let`으로 같은 이름 재선언 (타입 변경 가능) |
| 타입 추론 | 대부분 Rust가 알아서 판단 |
| 튜플 | 다른 타입 묶기, `tup.0`으로 접근 |
| 배열 | 같은 타입, 고정 길이, `arr[0]`으로 접근 |

> **Rust의 철학**: "실수를 컴파일 타임에 잡자" → 불변 기본, 타입 안전성
