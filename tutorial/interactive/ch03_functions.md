# Ch03: 함수 — 대화형 튜토리얼

---

## 1. 함수 기본 — fn = function

> **선생님**: Rust에서 함수는 `fn`으로 만들어요. **fn = function의 줄임말.**
>
> ```rust
> fn say_hello() {
>     println!("안녕!");
> }
>
> fn main() {
>     say_hello();
> }
> ```
>
> 근데 Rust가 다른 점이 하나 있어요. **매개변수에 반드시 타입을 써야 해요:**
>
> ```rust
> fn add(a: i32, b: i32) -> i32 {
>     a + b
> }
> ```
>
> - `a: i32, b: i32` — 매개변수는 타입 생략 불가
> - `-> i32` — 반환 타입
> - `a + b` — 세미콜론이 없으면 **이게 반환값**
>
> **왜 매개변수는 타입 추론 안 해주나?** → 함수는 다른 코드에서 호출되잖아요. 타입이 명시되어 있어야 "이 함수가 뭘 받고 뭘 주는지" 한눈에 보이니까요. 일종의 계약서예요.

### 💡 핵심
- `fn 이름(매개변수: 타입) -> 반환타입 { }`
- 매개변수 타입은 생략 불가 (함수 = 계약서)
- 마지막 줄에 세미콜론 없으면 그게 반환값

## 2. 세미콜론의 비밀 — 표현식 vs 구문

**🙋 학생**: 세미콜론을 붙이면 리턴되나?

> **선생님**: 반대예요!
>
> - `a + b` (세미콜론 없음) → **값을 반환함** ✅
> - `a + b;` (세미콜론 있음) → **반환 안 함, 컴파일 에러!** ❌
>
> ```rust
> fn add(a: i32, b: i32) -> i32 {
>     a + b      // 세미콜론 없음 → 반환됨
> }
>
> fn add2(a: i32, b: i32) -> i32 {
>     a + b;     // 세미콜론 있음 → 에러!
> }
> ```
>
> `return`을 써도 되지만, Rust 관례는 마지막 줄에서 세미콜론 빼고 반환. `return`은 함수 중간에서 일찍 빠져나올 때만 써요.

**🙋 학생**: 표현식하고 구문이라는게 정의가 어떻게 되는거지?

> **선생님**: 쉽게 말하면:
> - **표현식(expression)** = **값을 만들어내는 것**
> - **구문(statement)** = **실행만 하고 값은 없는 것**
>
> ```rust
> // 표현식 — 값이 나옴
> 5 + 3          // → 8
> add(1, 2)      // → 함수 반환값
>
> // 구문 — 값 없이 실행만
> let x = 5;     // 변수 선언은 구문
> ```
>
> 그래서 Rust에서 이런 것도 가능해요:
>
> ```rust
> let y = if true { 5 } else { 10 };  // if가 표현식이라 값을 변수에 넣을 수 있음!
> ```
>
> **세미콜론의 역할**: 표현식 뒤에 `;`를 붙이면 → "이 값 버려" → 구문이 됨.

### 💡 핵심
- 표현식 = 값을 만듦, 구문 = 실행만 함
- 세미콜론 = "이 값 버려"
- Rust는 표현식 중심 언어 (if도 표현식!)

## 3. 설계 철학과의 연결

**🙋 학생**: 설계 철학과도 관련이 있을까?

> **선생님**: 직접적으로 관련 있어요. Rust는 **expression-oriented language**(표현식 중심 언어)예요.
>
> ```rust
> // C++ 스타일 — 초기화 안 된 상태가 존재
> let msg;
> if score > 90 { msg = "A"; } else { msg = "B"; }
>
> // Rust 스타일 — 선언과 동시에 확정
> let msg = if score > 90 { "A" } else { "B" };
> ```
>
> **"불변이 기본"**이랑 연결됨: 변수 선언할 때 바로 값이 확정 → `mut` 불필요, 초기화 실수 불가능.
>
> 결국 같은 철학: **"컴파일 타임에 실수를 잡자"** → 불변 기본, 표현식 중심, 타입 명시. 다 연결됩니다.

**🙋 학생**: 그럼 값을 반환하는게 표현식이고 Rust는 가능하면 표현식으로 사용하고 세미콜론을 찍는다라고 이해하면 될까?

> **선생님**: 거의 맞는데 세미콜론이 반대!
> - 값을 **쓰고 싶으면** → 세미콜론 **안** 찍음
> - 그냥 **실행만 하고 싶으면** → 세미콜론 찍음

**🙋 학생**: 근데 if 문은 표현식인데 찍잖아 Rust에서.

> **선생님**: 날카로운 지적! 그건 `if` 표현식에 붙은 게 아니라 **`let` 구문에 붙은 거예요.**
>
> ```rust
> let msg = (여기가 표현식);
> //                        ^ 이 세미콜론은 let 구문의 끝
> ```
>
> **세미콜론은 항상 "문장의 끝"에 찍는 거고, 표현식 자체에 찍는 게 아닙니다.**

## Ch03 총정리

| 개념 | 한 줄 요약 |
|------|-----------|
| `fn` | function 줄임말, 함수 선언 |
| 매개변수 타입 | 반드시 명시 (함수 = 계약서) |
| `-> 타입` | 반환 타입 |
| 세미콜론 없음 | 값을 반환 (표현식) |
| 세미콜론 있음 | 값을 버림 (구문으로 변환) |
| `return` | 중간 탈출용에만 사용 (관례) |
| 표현식 중심 | if도 값을 만듦, 변수에 바로 대입 가능 |

> **Rust 설계 철학**: 표현식 중심 → 선언과 동시에 값 확정 → 불변 기본과 자연스럽게 연결
